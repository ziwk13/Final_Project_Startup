### Git 브랜치 전략 및 커밋 규칙

#### 브랜치 전략

- **`main`**: 최종 배포를 위한 브랜치.
- **`develop`**: 개발의 중심이 되는 브랜치. 기능 개발이 완료되면 `develop`으로 병합.
- **`개인이름`**: 기능 개발을 위한 개인 브랜치. (예: `kwonhyeongtaeg`)

#### 병합(Push) 규칙

- `develop` 브랜치로의 병합은 기능 단위로 완결된 상태에서만 가능.
- 모든 `develop` 브랜치로의 병합은 PR 요청하여 관리자의 코드 리뷰를 거친 후, 최종 확인 후 병합.
- `develop` 브랜치로의 병합 전 백엔드 코드 작성 시, 관련 테스트 코드를 반드시 작성하고 모든 테스트를 통과해야 함.
- 공통 라이브러리 및 리소스(JS, CSS 등) 추가/변경 시, 구동에 문제가 없음을 확인하고 전체 팀원에게 내용을 공유한 후 `develop` 브랜치에 반영.

#### 커밋 메시지 규칙

커밋 메시지는 해당 커밋의 내용을 명확하게 파악할 수 있도록 작성.

- **예시**:
    - `UserController - 로그인 매핑 추가 및 구현.`
    - `UserController - 로그인 매핑 추가 및 구현. / UserService - 사용자 로직 인터페이스 생성`

### 프로젝트 및 패키지 구조
기본적으로 도메인 구조를 이용한다.
```
project-root/
├── public/
├── src/
│   ├── api/                  # 공통 API 클라이언트 설정 (axios 등)
│   ├── assets/               # 공통 정적 파일
│   ├── components/           # 모든 도메인에서 재사용 가능한 공통 컴포넌트
│   │   ├── Button.js
│   │   └── Input.js
│   ├── features/             # 기능(도메인)별로 디렉토리 분리
│   │   ├── auth/             # '인증' 기능 관련 코드 집합
│   │   │   ├── api/          # 인증 API 호출 함수
│   │   │   ├── components/   # 인증 기능 내에서만 사용하는 컴포넌트 (LoginForm)
│   │   │   ├── hooks/        # 인증 관련 커스텀 훅 (useAuth)
│   │   │   ├── pages/        # 인증 관련 페이지 (LoginPage, SignupPage)
│   │   │   └── store/        # 인증 상태 슬라이스
│   │   └── mail/             # '메일' 기능 관련 코드 집합
│   │       ├── api/
│   │       ├── components/   # MailList, MailViewer 등
│   │       ├── hooks/
│   │       ├── pages/        # MailboxPage
│   │       └── store/
│   ├── hooks/                # 전역적으로 사용되는 커스텀 훅
│   ├── lib/                  # 외부 라이브러리 설정 또는 전역 유틸리티 (utils)
│   ├── provider/             # 전역 Context Provider 모음
│   ├── routes/
│   ├── store/
│   ├── styles/
│   ├── App.js
│   └── index.js
└── package.json
```

### 코딩 컨벤션
#### 컴포넌트 작성 규칙
- 컴포넌트 작성 시, 함수 선언식을 사용.

#### 데이터 요청 규칙
- 컴포넌트 내에서 데이터가 필요할 경우, useEffect 훅을 사용하여 데이터를 요청한다. 
- 라우터의 loader와 같이 렌더링 이전에 데이터를 가져오는 방식은 사용하지 않으며, 데이터 로직을 해당 데이터가 사용되는 컴포넌트 내에 위치시켜 응집도를 높인다.

### CSS 네이밍 규칙

CSS 스타일 간의 충돌을 방지하기 위해, 각자 담당하는 영역의 최상위 HTML 요소에 id를 부여하고, 해당 id를 부모 선택자로 활용. id의 값은 본인의 이니셜을 사용.

#### CSS 작성 예시
```css
/* 홍길동: hgd */
#hgd .header {
    background-color: blue;
}

/* 김영희: kyh */
#kyh .header {
    background-color: red;
}
```

#### HTML 작성예시
```html
<div id="hgd">
    <header class="header">
    </header>
</div>

<div id="kyh">
    <header class="header">
    </header>
</div>
```
